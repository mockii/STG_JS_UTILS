This Page

This page is temporary, and is located in the following directory: client/src/app/components/welcome.

The files are structured to help separate logical pieces from each other. Nested components should be located in the same directory.

Logo

Currently, the logo is 427px x 63px and is located at: client/src/assets/img/appLogo.png. Please keep the size the same however update/change the logo as appropriate.

Page Titles

The white bar at the top is a holder for page titles. These are optional and located in the stateProvider section of the module. This example page config looks like:

angular.module("myapp.welcome", [
    'myapp.welcome.controller'
])
    .config(['$stateProvider', function(stateProvider){
        stateProvider
            .state('welcome', {
                url: "/",
                templateUrl: "welcome/welcome.tpl.html",
                controller: "WelcomeController as welcomeController",
                data: {
                    pageTitle: "Welcome!"
                }
            });
    }]);

Navigation Control

The navigation bar is controlled in the application controller client/src/app/app.js and index.html client/src/index.html files.

appController.navItems = [
    {
        pageName: "Welcome",
        state: "/"
    }
];
<div stg-nav-bar class="page-header" menu-items="appController.navItems"></div>
This item accepts an array of objects, with pageName and state as the keys.


Super-lists will be added in the future (multi-dimensional array objects)

Building

You'll notice that your dist folder will have some additional files included into it. This is the result of integrating the STGWebUtils Client Libraries into the project. The package is installed via npm install which references the package in the src/client/package.json file.

During install, the src/client/gruntfile.js will reach into the src/client/node_modules folder, retrieve the appropriate assets (js, css, images, fonts, etc), and place them into the appropriate folders in the src/client/dist folder.
There are NO file checks in place to prevent overwriting an existing library asset file.
